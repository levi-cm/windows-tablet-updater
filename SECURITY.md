# Security Policy

## Reporting a vulnerability

Please report vulnerabilities privately through GitHub's **Security** tab and
its private vulnerability reporting form. Do not include exploit details,
credentials, device logs, or personal data in a public issue.

Include affected mode (`mit-admin` or `ohne-admin`), Windows version, updater
commit, reproduction steps, and impact. Reports are reviewed on a best-effort
basis; no response or remediation deadline is guaranteed.

Only the current `main` revision is supported. Before reporting, reproduce with
`-DryRun` when that can be done safely.
